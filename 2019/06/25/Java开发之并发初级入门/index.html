什么是并发

在操作系统中，并发，共享，虚拟，异步是操作系统的基本特征，其中并发是操作系统的最重要特征。

并行性是指两个或多个事件在同一时刻发生

并发性是指两个或多个事件在同一时间间隔内发生

在多道程序环境下，并发性是指在一段时间内宏观上有多个程序在同时运行，但在单处理机系统中每一时刻却仅能有一道程序执行，故微观上这些程序只能是分时地交替执行。

倘若在计算机系统中有多个处理机，则这些可以并发执行的程序便可被分配到多个处理机上，实现并行执行，即利用每个处理机来处理一个可并发执行的程序，这样，多个程序便可同时执行。

（收集几个专业名词：提高系统资源的利用率，增加系统的吞吐量 ）

进程与线程简述

  • 程序：通常的程序是静态实体(Passive Entity)，在多道程序系统中，它们是不能独立运行的，更不能和其它程序并发执行；

  • 进程：为使多个程序能并发执行，系统必须分别为每个程序建立进程(Process)。简单来说，进程是指在系统中能独立运行并作为资源分配的基本单位（此时尚未引入线程），它是由一组机器指令、数据和堆栈等组成的，是一个能独立运行的活动实体。（背诵，重点后一句话）；

  • 进程的缺点以及为什么引入线程：由于进程拥有自己的资源，故使调度付出的开销较大。通常在一个进程中可以包含若干个线程，它们可以利用进程所拥有的资源。

  • 线程与进程：通常都是把进程作为分配资源的基本单位，而把线程作为独立运行和独立调度的基本单位。

程序并发执行的特征：间断性；失去封闭性；不可再现性

以上，简单抄录了《操作系统》教材的几句关于并发，进程，线程的概念。我们所谓的Java并发编程其实质是指进行多线程的编程，接下来我们进入多线程的环境。

线程的生命周期

线程在生命周期内存在new（新建状态），ruannable（就绪状态），running(运行状态)，blocked（阻塞状态），dead（终止状态）五种状态。

具体来说：

  • new，新建状态，是线程被创建且未启动的状态。

    创建线程由三种方式：

    • 继承Thread
    • 实现Runnable
    • 实现Callable

  • runnable，就绪状态，是调用star()之后的状态；

  • running，运行状态，是run()正在执行时线程的状态（获得CPU执行时间片）

  • blocked，阻塞状态，进入此状态，有以下情况：

    • 同步阻塞：锁被其他线程占用
    • 主动阻塞：调用Thread的方法主动让出CPU执行权，如sleep(), join()等
    • 被动阻塞：执行了wait()

  • dead，终止状态，run()执行结束，或因异常退出的状态
