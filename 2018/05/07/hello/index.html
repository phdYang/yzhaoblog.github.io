这是整个网站的第一篇文章。我可能会写一下想法，计划，然后再交代一下建站的背景和使用的一些技术细节

开篇

这是2018年5月7日的一天，这照例应该是个普通的一天。

但这一天，一个新的网站就这样上线了。

对，所以，他注定不会是一个普通的一天的。

是吧，这应该是我，一个未来的起始。

我想，是时候，可以上线了。

建站技术概述

我有这个想法的时候，就去搜索比对了很多相关的模板，框架，以及现在成熟的一些技术方法。

最著名的方法可能属于WordPress了，一个前后台都包括的php博客架构，但我可能着实养不起一个服务器。

所以思来想去，还是选择了本站所使用Hexo + Github Pages的方式。但以后可能还是会迁移到真正的云服务器上的。

静待以后吧。

Hexo

Hexo是一个纯Html+CSS+JavaScript编写的一个纯静态页面。虽然官方提供了在线编辑博客的插件，但本质依然还是本地编写，再进行生成，部署的过程。

对于普通用户，他具有方便的配置文件可提供丰富的自定义样式，本站所呈现的效果都是通过配置文件来完成的，期间没有编写一行代码。

而对于前端高手来说，完全是一个高度自定义的博客框架，可以实现你想要实现的任何功能。

GitHub Pages

我们知道github是全球最大的同性交友网站。他本质上是一个代码托管的工具，当你苦恼于你的代码无处放置的时候，可以交给github。

因为他的上述特性，github也成了全球最大的开源项目分享基地，我们如雷贯耳的TensorFlow, PyTorch都是github的火爆项目。

基于代码托管的特性，github推出了github pages, 在这上面你可以显示你的个人简历，简单的页面，总之是没有数据库等后台代码存在的静态网站。

博客站

基于Hexo的静态页面框架特性和github pages的静态页面特性，二者成为了众多个人小型博客站的不二选择。

因为上述的技术选型，那么注定了我们的个人站访问速度受限，并且github本身在国外的原因，速度已经有些缓慢。所以，只适合个人静静的来作为一个安静的美男子使用。

同时，静态的页面必然少了许多花哨的功能，这里我们不得不借助于各种第三方插件来实现我们的小功能。

未来

公众号一直用来发布一些自己的平时胡乱写的东西，并且公众号不支持Markdown语法，对于技术类的文章支持不够友善，再加之自己对技术的不太了解。所以，我打算将大部分的技术类的文章会放在这个博客站上。

既然是自己的理解，那么难免有疏漏，偏驳，甚至是错误的观点，所以大家可以尽情的通过邮箱，或者直接使用留言来指出我的纰漏。

偶尔，也会放一些自己的感悟，随笔，小说等一些不成熟的小东西。稍微成熟一些的，我大都会放在公众号里面同大家一起分享。

所以，这个小站的性质，有点类似于自己的草稿纸吧。

如果有幸能得到各位的垂青，万分感激。

我的初衷，就是与大家共同进步!


另外，本网站只是本人的一个非常简陋的静态个人站，没有任何优化和防护措施，希望技术大佬放过小站一命！感谢不杀之恩！

欢迎大家加入洋仔的生活馆
